# 妖怪コレクション プロジェクト設計まとめ

**運営**: 合同会社妖怪屋（代表: 相田一成）
**ミッション**: 地域の未活用資産（夜・宴会場・神域）を「妖怪×エンタメ×IT」で再定義し、新たな観光価値と収益を創出する

---

## 1. ゲーム基本構造

### 1-1. 二陣営対立モデル

プレイヤーは以下の二陣営のいずれかに属する。

**祓い手陣営（秩序側）**
- 構成ロール: 陰陽師、祈祷師、巫女
- 目的: 妖怪を**封じる**
- 思想: 神道・仏教的世界観、調伏と鎮魂

**超自然派陣営（混沌側）**
- 構成ロール: 妖術師、山伏、呪術師
- 目的: 妖怪を**見つけて仲間にする / 解放する**
- 思想: 民間信仰、修験道、付喪神文化

※「払う」はお金を払うではなく **邪気を祓う** の意。

### 1-2. ポータル共通設計

- **ポータル（神社仏閣）は両陣営で共通**
- 同じ場所で目的が逆になる → PvP的緊張感
- 寺社側は陣営を問わず賽銭が落ちる構造（収益最大化）
- プレイヤーの陣営変更コストが低く、継続率向上

### 1-3. 賽銭による効果分岐

| 賽銭する陣営 | 効果 |
|------------|------|
| 祓い手陣営 | そのポータルでの**封印コスト減** |
| 超自然派陣営 | そのポータルでの**解放/使役コスト減** |

宗教的整合性: 両者とも「**鎮める**ための奉納」と説明統一。神道の荒魂/和魂の発想と相性が良い。

## 2. 賽銭システム設計

### 2-1. 金額設定

- **賽銭上限: 1回1,000円**
  - 賭博性・景品表示法リスクの回避
  - 健全性の担保（家族・未成年でも安心）
  - リアル賽銭の感覚と一致
  - 寺社への説明が容易

### 2-2. 推奨刻み

5円 / 50円 / 100円 / 500円 / 1,000円（リアル硬貨/紙幣ベース）

※45円（始終ご縁）等の語呂合わせも検討

### 2-3. 効果カーブ

logカーブで設計。5円でも効果はあるが、1,000円が最も効率的。ただし200倍差にはしない（賽銭文化的に自然な範囲）。

## 3. 収益分配モデル

### 3-1. 法的建付け

**OK: ユーザーが寺社に寄付する。妖怪屋は決済・プラットフォーム提供の対価として手数料を取る。**

NG: ユーザーが妖怪屋に払う → その一部を寺社に分配する（宗教法人非課税枠が使えない＋優良誤認リスク）

### 3-2. 配分案

| 配分先 | 比率 |
|-------|------|
| 寺社 | 70〜85% |
| 妖怪屋（プラットフォーム手数料） | 10〜20% |
| 決済代行（Stripe等） | 3〜4% |

参考: GoFundMe 2.9%、CAMPFIRE 17%、ふるさとチョイス（自治体課金型）。**15%前後**が業界相場として説明しやすい。

### 3-3. ユーザー透明性（必須）

賽銭画面で内訳を明示:

```
賽銭 1,000円
├ ◯◯神社へ 850円
├ 決済手数料 30円
└ プラットフォーム利用料 120円
```

### 3-4. 寺社への説明話法

「賽銭の◯%を抜く」ではなく「**集客・アプリ開発・運用保守を一括で巻き取る対価**として◯%」と説明。IT導入の業務委託料として理解しやすい構造。

## 4. ユーザー恩恵設計（賽銭インセンティブ）

実装候補:

1. **ご利益バフ** — 妖怪遭遇率アップ、レア妖怪出現、捕獲/封印/解放成功率＋（神社ごとに固有効果）
2. **デジタル御守の発行** — 賽銭額に応じたインベントリアイテム＋バフ効果
3. **限定妖怪の解放** — 寺社の伝承に紐づく固有妖怪（伏見稲荷→九尾、出雲→八岐大蛇等）
4. **おみくじガチャ** — 賽銭500円でアプリ内おみくじ、結果でレア確定チケット
5. **奉納ランキング** — 月間ランキング、上位者に称号・限定スキン（氏子文化の現代版）
6. **絵馬機能** — 賽銭時に願い事＋スタンプ、他ユーザーと共感ボタンで応援
7. **祓い・厄落とし** — 「祟り属性」妖怪を賽銭で浄化（祓い手陣営向け）
8. **遠隔参拝** — 物理移動できない人の推し神社賽銭（コロナ後需要、海外在住日本人需要）

設計思想: ゲーム課金は「自分への投資」、賽銭は「寺社への貢献」という体験設計。同じ金額でも罪悪感が減り満足度が上がる。バフは時間経過で切れる → 自然なリピート発生。

## 5. 寺社との関係構築戦略

### 5-1. 大原則: 個別許諾必須

- 包括契約・神社本庁経由の一括は構造的に不可能
- 宗教法人ごとに事情が全然違う
- ポケモンGO訴訟前例あり、無許可ポータル化は訴訟リスク
- 賽銭が絡むと寺社側の責任が重くなる
- 代替わりリスクに備え「個人」ではなく「法人」と契約

### 5-2. 段階展開戦略

**フェーズ1: 旗艦寺社10〜30社と個別深掘り契約**
- 妖怪伝承が強い場所を選定（出雲、大江山、遠野、境港周辺、京都百鬼夜行系等）
- 書面契約、レベシェア・夜間拝観・限定御朱印まで作り込む
- **会津での1社目**: 相田代表の地元コネクションを活かす旗艦事例化

**フェーズ2: 標準契約書テンプレ＋簡易許諾**
- 旗艦事例ベースの標準契約で中規模寺社100〜500社展開

**フェーズ3: 申請プラットフォーム化**
- 寺社側から「うちもポータルにしてほしい」と申請できるWebフォーム
- 簡易審査＋電子契約

### 5-3. アプローチルート

- **神社系**: 神社本庁の地方支部、各県神社庁に「相談」スタンスで
- **仏教系**: 宗派の宗務庁単位（観光寺院多い宗派：臨済宗、天台宗、真言宗）
- **自治体経由**: 市町村観光課を巻き込み、地方創生×ITの看板で動く
- **既存妖怪縁の自治体**: 境港（鳥取）、遠野（岩手）、福崎（兵庫）、大江山（京都）

### 5-4. 寺社収益化メニュー（許諾止まりにしない）

1. レベニューシェア（文化財保護協力金/寄進名目で非課税枠活用）
2. 夜間拝観チケット抱き合わせ
3. 限定御朱印・授与品コラボ
4. 法要・祈祷送客モデル（祈祷料5,000〜10,000円）
5. 縁日・宴会場としての境内貸し（妖怪屋の本業活用）
6. ストーリー監修料・伝承提供料（神職を妖怪学アドバイザー委嘱）
7. 寄進プラットフォーム化（電子賽銭）
8. 共同事業体（LLP）化（重点寺社）

**推奨組み合わせ: 3＋4＋5**（寺社の既存ビジネスに乗せる形が宗教法人会計上クリーン）

## 6. リスク管理

### 6-1. 法務・権利

- 個別書面許諾＋宗教法人との契約（最優先）
- 既存IP（水木プロ、自治体登録商標）との非衝突確認
- 改正個人情報保護法・GPS追跡の同意取得
- 「賽銭ガチャ」設計は景表法・賭博罪グレー → 弁護士相談マスト

### 6-2. 寺社サイドの摩擦

- 参拝者と遊戯目的ユーザーの動線・マナー衝突
- 御朱印文化との誤認防止
- 檀家・氏子総会の合意（宮司/住職だけの口約束は危険）

### 6-3. 安全・地域

- 夜間運営の保険（PL/イベント保険）
- 私有地・参道通行権、夜間立入の境内使用許可
- 地元警察・近隣住民への事前通知

### 6-4. レピュテーション

- 「神聖な場所の商業利用」批判
- 戦没者慰霊系・事故/死亡があった寺社は地雷
- インバウンド客のマナー違反時の運営責任

### 6-5. 事業継続

- 宮司/住職代替わり対応 → 法人契約・3〜5年契約
