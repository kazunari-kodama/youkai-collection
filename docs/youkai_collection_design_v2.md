# 妖怪コレクション プロジェクト設計 v2

**運営**: 合同会社妖怪屋（代表: 相田一成）
**ミッション**: 地域の未活用資産（夜・宴会場・神域）を「妖怪×エンタメ×IT」で再定義し、新たな観光価値と収益を創出する
**バージョン**: v2（バトルなし陣取り型に再設計、MVP定義、可視化システム追加）

---

## 0. 用語定義

| 用語 | 意味 |
|------|------|
| ポータル | 神社仏閣等の実在拠点。プレイヤーが現地で作用する対象 |
| 祓う | 妖怪を封じる行為。祓い手陣営の主アクション |
| 契約 | 妖怪を仲間にする行為。超自然派陣営の主アクション |
| 賽銭 | アプリ内で寺社に行う電子寄付。ゲーム内バフを伴う |
| 陣営 | プレイヤーの所属。祓い手 / 超自然派の2系統 |
| JOB | 陣営内の役職。MVPは陰陽師・呪術師の2職 |
| 払う | 邪気を**祓う**の意。お金を払うではない |

## 1. ゲーム基本構造

### 1-1. ジャンル

**位置情報ベースの陣取りゲーム（バトルなし）**

- 戦闘要素は実装しない
- プレイヤー間の対立は「**蓄積競争**」で表現
- 寺社への説明が容易：「参拝と探索のゲームで戦闘要素は一切ありません」
- 客層が広い（高齢層・女性層・ファミリー層も入れる）

### 1-2. 二陣営対立モデル

**祓い手陣営（秩序側）**
- 思想：神道・仏教的世界観、調伏と鎮魂
- 目的：妖怪を**祓う**（封じる）
- MVP JOB：陰陽師
- プレミアム JOB：神主、祈祷師/巫女

**超自然派陣営（混沌側）**
- 思想：民間信仰、修験道、付喪神文化
- 目的：妖怪と**契約する**（仲間にする）
- MVP JOB：呪術師
- プレミアム JOB：山伏、妖術師

### 1-3. ポータル共通設計

- **ポータル（神社仏閣）は両陣営で共通**
- 同じ場所で目的が逆 → 緊張感
- 寺社側は陣営問わず賽銭が落ちる → 収益最大化
- プレイヤーの陣営変更コスト低 → 継続率向上

### 1-4. 賽銭による効果分岐

| 賽銭する陣営 | 効果 |
|------------|------|
| 祓い手陣営 | そのポータルでの**封印（祓う）コスト減** |
| 超自然派陣営 | そのポータルでの**契約コスト減** |

宗教的整合性：両者とも「**鎮める**ための奉納」と説明統一。神道の荒魂/和魂の発想と相性が良い。

## 2. 陣取りメカニクス

### 2-1. 基本動作

陣取りで発生する基本アクション：

1. **獲得** — ポータルで祓う / 契約する
2. **維持** — 自陣営状態を保つ（賽銭・参拝で霊力補充）
3. **奪取** — 相手陣営ポータルを自陣営に塗り替え
4. **連結** — ポータル同士を繋ぎ領域形成（陣形ボーナス）
5. **発見** — 新規ポータル、レア妖怪の発掘

### 2-2. 奪い合いの仕組み（バトルなし版）

**蓄積競争モデル**

- ポータルは「祓い側スコア」「契約側スコア」を常時両方持つ
- プレイヤーが現地で作用するごとに自陣営スコア加算
- 一定差で陣営支配状態が確定
- 賽銭・アイテム使用でもスコア加算可能

**ポータル劣化と更新**

- ポータルの霊力は時間経過で減衰
- 自陣営プレイヤーの参拝・賽銭で補充
- 霊力ゼロ → 相手陣営に奪われやすい状態
- 誰も来ない場所は自然にリサイクル可能

**作用上限**

- 1日あたりプレイヤーが作用できるポータル数に上限
- リアルに足を運ぶ必要 → 地方創生に直結

### 2-3. 陣形効果

- JOBの能力が地図上で重なる/連なると効果増幅
- 支配することの経済的メリット：自陣営エリアの妖怪出現率アップ、賽銭バフ強化等
- 奪う動機は戦闘の楽しさではなく**経済効果**

## 3. JOB設計

### 3-1. MVP: 陰陽師 vs 呪術師

この2職は機能的に綺麗な対称関係。

| 項目 | 陰陽師 | 呪術師 |
|------|--------|--------|
| 陣営 | 祓い手 | 超自然派 |
| 主機能 | 結界（陣形形成） | 妖怪駐留（領域支配） |
| 行動様式 | 幾何学的・戦略的 | 収集的・蓄積的 |
| 対極性 | 妖怪を遠ざける | 妖怪と契約する |

地図上で**結界 vs 妖怪駐留**という見た目も挙動も違う2レイヤーが拮抗。

### 3-2. 陰陽師の能力

**主機能：結界による広域支配**
- 3つ以上のポータルを直線/三角で結ぶと「結界陣」発動
- 内側のポータルは契約側から奪われにくくなる
- 結界はランクに応じて最大辺長が拡大

**基本スキル**

| スキル名 | 種別 | 概要 |
|---------|------|------|
| 陰陽眼 | パッシブ | 妖怪の気配を感知できる範囲が拡大 |
| 真名識別 | アクティブ | 同じ妖怪を繰り返し封印すると「真名」が解放される。真名を知ると封印コスト大幅減・式神化解放 |
| 封印符 | アクティブ | ポータルで妖怪を祓う基本アクション |
| 結界陣 | アクティブ | 3点以上のポータルを結んで結界を形成 |

**固有スキル：飛翔式神（ひしょうしきがみ）**

術力を消費して式神を遠隔地へ飛ばし、物理的に行けない場所の妖怪を封じる。

- 射程上限なし
- 飛行速度：1km = 10分（約6km/h、早歩き相当）
- 発射位置と目標ポータルを結ぶ直線を飛行
- 飛行中の式神は地図上に自分のみに表示（他プレイヤーには見えない）
- 飛行中のスロットが全て埋まると次の式神を発射できない
- 位置計算はクライアント側で補間（発射位置・目標位置・発射時刻・速度から算出）

| ランク | 同時飛行スロット |
|-------|--------------|
| C（陰陽生） | 1体 |
| B（陰陽師） | 2体 |
| A（陰陽博士） | 3体 |
| S（陰陽頭） | 4体 |
| SS（大陰陽師） | 5体 |

| 距離 | 到達時間 |
|------|---------|
| 1 km | 10分 |
| 5 km | 50分 |
| 20 km | 約3時間 |
| 50 km | 約8時間（寝てる間に到達） |
| 100 km | 約17時間（翌日） |

封印成功率は真名識別と連動：
- 真名を知らない妖怪：成功率 40%
- 真名を知っている妖怪：成功率 90%

### 3-3. 呪術師の能力

**主機能：契約妖怪の駐留**
- 契約した妖怪をポータルに駐留させ陣営支配を強化
- 駐留中の妖怪が陰陽師の結界に対する対抗力に
- 駐留可能ポータル数はランクで増加

**固有能力：百鬼夜行**
- 保有妖怪を一斉召喚（範囲効果）
- 発動回数制限あり
- ランクで規模拡大

### 3-4. MVP時の数値感（要調整）

- 結界は3ポータル最小、1人で組める設計
- 妖怪駐留は1ポータル1体から有効
- プレイヤー人口が少ない時期でも遊べる調整

### 3-5. プレミアム JOB（v1.1以降）

実装大変なため後回し。プレミアム機能として段階解放。

**祓い手陣営**
- **神主**：拠点防衛特化（氏神神社の絶対防衛、賽銭システムと連動）
- **祈祷師/巫女**：広域メンテ特化（祓詞による一括浄化）

**超自然派陣営**
- **山伏**：探索特化（新規ポータル発掘、自然エリア特化）
- **妖術師**：遠隔工作特化（呪詛による事前弱体化）

**設計思想：機能が違うだけで強さは同じ**
- プレミアム職が「強すぎる」P2Wにならないよう厳守
- 陰陽師・呪術師でも全機能アクセス可能、ただし一点特化はできない

## 4. ランク制

### 4-1. ランク構造

各JOBで C → B → A → S → SS の5段階。

### 4-2. JOB別ランク名（位階）

歴史実在の位階を採用し「本物感」を演出。

**陰陽師**（律令制陰陽寮の役職ベース）
C: 陰陽生 → B: 陰陽師 → A: 陰陽博士 → S: 陰陽頭 → SS: **大陰陽師**

**呪術師**（古代日本の呪術官職／中国渡来呪術ベース）
C: 呪い子 → B: 呪術師 → A: 蠱師 → S: 呪禁師 → SS: **大呪師**

**プレミアム職（参考）**
- 神主：直階 → 権正階 → 正階 → 明階 → **浄階**（神社本庁の実階位）
- 祈祷師/巫女：物忌 → 神和ぎ → 巫覡 → 御杖代 → **斎宮**
- 山伏：新客 → 度衆 → 正先達 → 大先達 → **大行満**
- 妖術師：外法者 → 妖術師 → 幻術師 → 鬼道師 → **大妖導師**

### 4-3. 昇格条件

陣取り貢献度ベース（戦闘なしのため平和的指標）。

| JOB | 昇格主要条件 |
|-----|------------|
| 陰陽師 | 形成した結界陣の総面積／日数＋式神運用実績 |
| 呪術師 | 駐留妖怪の総稼働ポータル日数＋契約成功数 |

### 4-4. SS級の特別感

- 全国で各JOBあたりSS級は7〜10人まで（人数制限）
- SS昇格には**儀式クエスト**（特定神社での集会等、リアルイベント連動）
- 称号がプロフィールに常時表示＋限定ビジュアル
- 年1回の**奉納大祭**でSS級同士が交流（妖怪屋主催イベント連動）

→ 妖怪屋本業（イベント運営）が公式の名誉システムに直結。

## 5. 勢力可視化システム

### 5-1. 設計思想

「**一目で今どっちが優勢か分かる**」のは陣取りゲームの命。
SplatoonのナワバリメーターやIngressのスキャナー画面と同等の重要度。

### 5-2. 全体勢力メーター（画面上部常設）

**基本表示**
```
[祓 ████████░░░░ 契]
     58%      42%
   12,847    9,231
```

**表示要素**
- 全国累計の祓った数 / 契約した数
- 比率（％ + 塗り分け）
- 自分の貢献分（点滅/ハイライト）
- 差分の動き（前日比、過去1時間など）

**色設計**
- 祓い側：白〜青系（清浄・神道）
- 契約側：紫〜赤系（呪術・闇）
- 拮抗時（45〜55%）：黄金色グラデーション → **百鬼夜行モード**突入の合図

### 5-3. 地域別勢力メーター（地図画面）

ズームレベルで表示単位が切り替わる：

| ズーム | 表示単位 |
|--------|---------|
| 全国 | 日本全体 |
| 広域 | 地方ブロック（東北・関東等） |
| 中域 | 都道府県 |
| 近域 | 市区町村 |
| 詳細 | ポータル単位 |

→ 「自分の地元では祓い側が劣勢、頑張ろう」というローカル動機が発生。

### 5-4. 妖怪別メーター（個体ページ）

各妖怪のページに、世界全体での祓われ数 vs 契約数を表示。

**例：九尾の狐ページ**
```
妖怪名：九尾の狐
出現地：伏見稲荷ほか

[祓 ███████████░░░░ 契]
        342  /  198
     祓われた  契約された

希少度：★★★★☆
あなたの実績：祓 3 / 契約 1
```

**効果**
1. 「未踏感」：契約数1桁の妖怪は契約勢にとって貴重
2. メタ攻略情報：契約困難度が一目で分かる
3. コレクション欲：祓われすぎてる妖怪は契約勢が保護したくなる
4. 祓い勢のモチベ：契約されすぎてる妖怪は祓いたくなる

→ **妖怪ごとに陣営対立の物語が発生する**。

### 5-5. リアルタイム流入アニメーション

メーターが動く瞬間、画面右上に通知：
```
↑ 京都府で祓われました（+1）
↓ 岩手県で契約されました（+1）
```

「今この瞬間も戦いが起きてる」感を演出。

### 5-6. 動的称号システム

祓われ数＋契約数の集計で自動付与：

| 状態 | 称号 |
|------|------|
| 総数が極めて少ない | 幻の妖怪 |
| 祓われすぎ | 絶滅危惧種 |
| 契約されすぎ | 飼い慣らされし者 |
| 均衡 | 争奪の的 |

### 5-7. 各種ランキング

- 最も祓われた妖怪 TOP10
- 最も契約された妖怪 TOP10
- 最も拮抗した妖怪 TOP10 ← 物語性が高い
- プレイヤーの貢献度ランキング（地域別/全国）

### 5-8. 個人ダッシュボード

```
あなたの実績
├ 総祓数：247（全国順位 1,234位）
├ 総契約数：18（全国順位 8,901位）
├ 陰陽師ランク：奥伝（A）
└ 貢献度メーター
   [祓 ███████████░ 契]
      93%      7%
   → あなたは「祓い派」です
```

→ 自分の偏りが可視化される。**たまには反対側も**という動機にも。

### 5-9. 拮抗時特別演出（百鬼夜行モード）

全国メーターが49〜51%の超拮抗状態 → 自動発令：

- 「百鬼夜行発令」の画面エフェクト
- 全プレイヤーに緊急通知
- 一定時間、両陣営にバフ（祓い/契約効果アップ）
- レア妖怪の出現率アップ

→ 運営側で意図的に拮抗演出を起こせる ＝ イベント設計と相性◎

### 5-10. 実装上の注意

**数値の母数問題（MVP期）**
- プレイヤー少ない時期は数値が小さく見栄えが悪い
- 対策：累計型でリセットしない / 比率を主表示 / 運営シードデータ投入

**チート対策**
- 1日の作用上限
- 位置情報の整合性チェック
- 異常増加にはサーバー側で補正

**負荷対策**
- 全国メーターは5〜10分ごと更新で十分
- 地域別はキャッシュ活用

**心理的設計**
- 大差時は「あと20体で逆転！」と僅差を煽る
- 個人貢献を強調
- 全国で劣勢時はローカル数値を前面に出す

## 6. 陣営均衡メカニクス（劣勢対策）

### 6-1. 課題

二陣営構造では片方が劣勢になると、もう片方をやるメリットがなくなる構造的死因がある。Ingress初期と同じ問題。

### 6-2. 対策

**ラバーバンド（劣勢ボーナス）**
- 劣勢陣営は祓い/契約効率が1.5倍
- 劣勢陣営でしか出ない限定妖怪
- 「逆境ボーナス」称号やコスメ報酬
- 注意：あからさますぎると「わざと負ける」メタが発生 → 月別リセット推奨

**共生型インセンティブ**
- 妖怪が暴れすぎると地域災害発生 → 両陣営にデメリット
- 両陣営の賽銭比率が拮抗だと地域ボーナス発生
- 対立してるのに互いを呼び込む構造

**第三勢力**
- 外来妖怪（西洋悪魔・大陸の妖怪）：両陣営共通の敵
- 荒ぶる神：地域全体に脅威 → 両陣営協力で対処
- 中立勢力（付喪神/精霊系）

**地域陣営の独立**
- プレイヤー陣営とは別にエリア単位の支配状態
- 「敵地」での活動報酬倍率
- 全国劣勢でもローカル逆転の余地

**シーズン制**
- 月替わりで「祓いの月」「百鬼夜行の月」交代
- 季節イベント（節分=祓い強化、お盆=契約強化）
- 妖怪屋本業のイベント運営と直結

## 7. 賽銭システム

### 7-1. 設計思想

ゲーム課金は「自分への投資」、賽銭は「寺社への貢献」という体験設計。同じ金額でも罪悪感が減り満足度が上がる。

### 7-2. 金額設定

**賽銭上限：1回1,000円**
- 賭博性・景表法リスク回避
- 健全性担保（家族・未成年でも安心）
- リアル賽銭の感覚と一致
- 寺社への説明が容易

### 7-3. 推奨刻み

5円 / 50円 / 100円 / 500円 / 1,000円（リアル硬貨/紙幣ベース）
※45円（始終ご縁）等の語呂合わせも検討

### 7-4. 効果カーブ

logカーブで設計。5円でも効果はあるが、1,000円が最も効率的。ただし200倍差にはしない（賽銭文化的に自然な範囲）。

### 7-5. 制限設計

- 1ポータル/1日の賽銭回数制限
- クールタイム設定
- 月間上限 or 警告システム
- マルチアカウント対策（1端末1陣営縛り、陣営変更クールタイム）

### 7-6. ユーザー恩恵

1. **ご利益バフ** — 妖怪遭遇率・捕獲/封印/契約成功率アップ
2. **デジタル御守** — 賽銭額に応じたインベントリアイテム
3. **限定妖怪解放** — 寺社伝承固有妖怪
4. **おみくじガチャ** — 賽銭500円でレア確定チケット
5. **奉納ランキング** — 月間ランキング（氏子文化の現代版）
6. **絵馬機能** — 願い事＋スタンプ、共感ボタン
7. **祓い・厄落とし** — 祟り属性妖怪の浄化（祓い手向け）
8. **遠隔参拝** — 推し神社賽銭（海外在住日本人需要含む）

### 7-7. バフは時間経過で切れる

リピート賽銭が自然発生する設計。月課金より健全で、月課金より総額大きくなる可能性。

## 8. 収益分配モデル

### 8-1. 法的建付け

**OK**：ユーザーが寺社に寄付する。妖怪屋は決済・プラットフォーム提供の対価として手数料を取る。
**NG**：ユーザーが妖怪屋に払う → その一部を寺社に分配（宗教法人非課税枠が使えない＋優良誤認リスク）

### 8-2. 配分案

| 配分先 | 比率 |
|-------|------|
| 寺社 | 70〜85% |
| 妖怪屋（プラットフォーム手数料） | 10〜20% |
| 決済代行（Stripe等） | 3〜4% |

参考：GoFundMe 2.9%、CAMPFIRE 17%。**15%前後**が業界相場として説明しやすい。

### 8-3. ユーザー透明性（必須）

賽銭画面で内訳を明示：
```
賽銭 1,000円
├ ◯◯神社へ 850円
├ 決済手数料 30円
└ プラットフォーム利用料 120円
```

### 8-4. 寺社への説明話法

「賽銭の◯%を抜く」ではなく「**集客・アプリ開発・運用保守を一括で巻き取る対価**として◯%」。IT導入の業務委託料として理解しやすい構造。

## 9. 寺社との関係構築戦略

### 9-1. 大原則：個別許諾必須

- 包括契約・神社本庁経由の一括は構造的に不可能
- 宗教法人ごとに事情が違う
- ポケモンGO訴訟前例あり、無許可ポータル化は訴訟リスク
- 賽銭が絡むと寺社側の責任が重くなる
- 代替わりリスクに備え「個人」ではなく「法人」と契約

### 9-2. 段階展開戦略

**フェーズ1：旗艦寺社10〜30社と個別深掘り契約**
- 妖怪伝承が強い場所を選定（出雲、大江山、遠野、境港周辺、京都百鬼夜行系等）
- **会津での1社目**：相田代表の地元コネクション活用

**フェーズ2：標準契約書テンプレ＋簡易許諾**
- 旗艦事例ベースで中規模寺社100〜500社展開

**フェーズ3：申請プラットフォーム化**
- 寺社側からの申請受付Webフォーム
- 簡易審査＋電子契約

### 9-3. アプローチルート

- **神社系**：神社本庁地方支部、各県神社庁に相談スタンスで
- **仏教系**：宗派の宗務庁単位（観光寺院多い宗派：臨済宗・天台宗・真言宗）
- **自治体経由**：市町村観光課を巻き込み、地方創生×ITの看板で
- **既存妖怪縁の自治体**：境港、遠野、福崎、大江山

### 9-4. 寺社収益化メニュー

1. レベニューシェア（文化財保護協力金/寄進名目）
2. 夜間拝観チケット抱き合わせ
3. 限定御朱印・授与品コラボ
4. 法要・祈祷送客モデル
5. 縁日・宴会場としての境内貸し
6. ストーリー監修料・伝承提供料
7. 寄進プラットフォーム化（電子賽銭）
8. 共同事業体（LLP）化

**推奨組み合わせ：3＋4＋5**（寺社の既存ビジネスに乗せる形が会計上クリーン）

## 10. リスク管理

### 10-1. 法務・権利
- 個別書面許諾＋宗教法人との契約（最優先）
- 既存IP（水木プロ、自治体登録商標）との非衝突確認
- 改正個人情報保護法・GPS追跡の同意取得
- 「賽銭ガチャ」設計は景表法・賭博罪グレー → 弁護士相談マスト

### 10-2. 寺社サイドの摩擦
- 参拝者と遊戯目的ユーザーの動線・マナー衝突
- 御朱印文化との誤認防止
- 檀家・氏子総会の合意（宮司/住職だけの口約束は危険）

### 10-3. 安全・地域
- 夜間運営の保険（PL/イベント保険）
- 私有地・参道通行権、夜間立入の境内使用許可
- 地元警察・近隣住民への事前通知

### 10-4. レピュテーション
- 「神聖な場所の商業利用」批判
- 戦没者慰霊系・事故/死亡のあった寺社は地雷
- インバウンド客のマナー違反時の運営責任

### 10-5. 事業継続
- 宮司/住職代替わり対応 → 法人契約・3〜5年契約
